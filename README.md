# CalSnap 📱 – AI Powered Calorie Tracker

## Overview
CalSnap lets users **snap a photo of their meal**, automatically recognises the food, fetches nutrition facts from OpenFoodFacts and returns a short health advice generated by LLaMA 3.  
Free tier = 5 scans per day, Premium = unlimited + analytics.

| Layer | Tech |
|-------|------|
| Front‑end | React Native (Expo) |
| Backend | Node.js + Express |
| Database / Auth / Storage | Supabase (free tier) |
| Food detection | YOLOv8 (Replicate) |
| OCR | Tesseract.js |
| Nutrition lookup | OpenFoodFacts API |
| Reasoning | LLaMA 3 (Replicate) |
| CI / CD | Codemagic (free) |
| Payments | Stripe (test) |

## Prerequisites
* **Node 18+** & **npm**
* **Expo CLI** – `npm i -g expo-cli`
* **Supabase** account (free tier)
* **Replicate** account (free GPU minutes) – you’ll need an API token.
* **Stripe** test account (optional for premium subscription)
* **Git** (to clone the repo)

## 1️⃣ Set up Supabase
1. Create a new Supabase project.  
2. In **SQL editor**, run the file `database/schema.sql`.  
3. Enable **Row Level Security** for all tables and add the policies that come with the script.  
4. Create a **Storage bucket** called `food-images` and set it to **Public**.  
5. Copy the **API URL**, **ANON PUBLIC** key and **SERVICE ROLE** key – you’ll need them for the env files.

## 2️⃣ Set up Replicate (YOLOv8 + LLaMA 3)
1. Sign‑up at https://replicate.com, get a **API token**.  
2. Find a YOLOv8 model (e.g. `ultralytics/yolov8`) – copy its **version hash** (you’ll see something like `c2c5e99531b2...`).  
3. Find a LLaMA 3 model (e.g. `meta-llama/Meta-Llama-3-8B`) – copy its **version hash**.  
4. Add the three values to the backend `.env` (see the example below).
